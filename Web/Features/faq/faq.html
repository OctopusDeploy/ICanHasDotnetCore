The new tool stack

The new .NET tool stack includes the dotnet command line tool, xproj project structure and JSON configuration.

It is possible to take most existing applications, targeting .NET 4.5.1 or later and convert them to use this new structure. This is done by creating a new .NET Core project, editing the project.json file to change the frameworks section to target net451 (example).

This website (source) uses the new tooling, but targets .NET 4.6.1 as not all our dependencies have been ported to .NET Standard. The new tooling supports targeting multiple platforms, so each project can be gradually be converted to also target .NET Standard when appropriate.

.NET Standard

.NET Standard Library is a set of APIs that are intended to be available on all .NET runtimes. Applications targeting .NET Standard should run on all platforms that implement that standard. See the refer to the .NET Standard Library documentation.

How does this work?

We read your packages.config file and recursively lookup up those packages and their dependencies on nuget.org. We then look at whether there are any .NETStandard dependencies. Since what was the base class library is now referenced via nuget, the majority of .NETStandard assemblies reference at least one package.

Do you keep any uploaded information?

Yes, but in a non-identifying way. We keep a count of how many time a packages found on nuget.org is appears in a result set. We do not track the names of packages that were not recognised, as these would typically be from internal nuget feeds.
